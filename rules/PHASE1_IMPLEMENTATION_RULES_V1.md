## 时间语义

1. 所有输入先解释为绝对时间，再转换为北京时间（UTC+08:00）。
2. 字符串输入必须包含 `Z` 或显式 UTC 偏移；无时区字符串无效。
3. 运气年 `Y` 的范围是 `Y` 年大寒实际交节时刻至 `Y+1` 年大寒实际交节时刻。
4. 边界采用左闭右开：交节时刻本身属于新运气年或新六步。
5. 节气时刻必须由 `CalendarProvider` 提供，默认适配 `tyme4ts`；禁止用固定公历日期替代。

### 客气序列

客气按以下三阴三阳序列循环：

```text
厥阴风木 → 少阴君火 → 太阴湿土 → 少阳相火 → 阳明燥金 → 太阳寒水
```

将当年司天置于三之气，其余客气按序列前后循环；由此终之气必须等于在泉。实现仍须用测试直接验证“三之气 = 司天”和“终之气 = 在泉”，不得只依赖推论。

## 客主关系

六气所属五行：

| 六气 | 五行 |
| --- | --- |
| 厥阴风木 | 木 |
| 少阴君火 | 火 |
| 太阴湿土 | 土 |
| 少阳相火 | 火 |
| 阳明燥金 | 金 |
| 太阳寒水 | 水 |

五行相生：`木→火→土→金→水→木`。

五行相克：`木→土→水→火→金→木`。

关系判定按以下优先级，必须且只能返回一个值：

1. 气名完全相同：`SAME_QI`
2. 五行相同但气名不同：`SAME_ELEMENT_DIFFERENT_QI`
3. 主气生客气：`HOST_GENERATES_GUEST`
4. 客气生主气：`GUEST_GENERATES_HOST`
5. 主气克客气：`HOST_CONTROLS_GUEST`
6. 客气克主气：`GUEST_CONTROLS_HOST`

君火与相火必须保留为不同气名；二者只在五行层面同属火。
